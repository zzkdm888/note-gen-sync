```markdown
# SOP 详细文档

## 流程阶段详情

### 阶段一：目标侦察与规划 (Reconnaissance & Planning)
1.  **1.1 目标定义与信息收集**
    *   **被动侦察:**
        - 使用公开来源情报 (OSINT) 搜集目标信息。
        - => 查询域名注册信息 (Whois), DNS记录, Google Hacking (Dorks)。
        - => 分析社交媒体、招聘网站、公司财报，获取技术栈、员工姓名、组织结构。
    *   **主动侦察:**
        - 直接与目标网络交互以获取信息。
        + **网络扫描:** 使用 `nmap`, `masscan` 等工具扫描开放端口、服务及版本。
        + **子域名爆破:** 使用 `sublist3r`, `amass` 等工具发现目标相关的子域名。
        - **目录扫描:** 使用 `gobuster`, `dirb` 扫描网站隐藏目录和文件。

2.  **1.2 漏洞分析与攻击向量选择**
    *   **漏洞识别:**
        - 将扫描到的服务版本与公开漏洞数据库 (CVE, Exploit-DB) 进行比对。
        - 使用漏洞扫描器如 `Nessus`, `OpenVAS` 对目标进行系统性扫描。
    *   **攻击向量决策:**
        - => 基于发现的漏洞和目标防御水平，选择成功率最高、最隐蔽的攻击路径。
        - **技术向量:** 软件漏洞利用、Web应用攻击 (SQLi, XSS, RCE)、配置错误。
        - **社工向量:** 钓鱼邮件 (Phishing)、鱼叉式钓鱼 (Spear Phishing)。

### 阶段二：武器构建与投送 (Weaponization & Delivery)
1.  **2.1 工具/载荷开发与配置**
    *   **载荷 (Payload) 生成:**
        - 使用 `Metasploit Framework (msfvenom)` 生成定制化的Shellcode或后门程序。
        - 编写或修改现有恶意软件，使其具备免杀 (FUD - Fully Undetectable) 特性，绕过AV/EDR。
    *   **利用代码 (Exploit) 准备:**
        - 获取或购买现有漏洞利用代码。
        - 针对特定环境修改或自行开发0-day/1-day利用代码。

2.  **2.2 投送机制准备**
    *   **钓鱼邮件构建:**
        - 注册与目标相似的域名 (Typosquatting)。
        - 制作高仿真的邮件模板和登陆页面。
        - 将载荷嵌入附件或链接中。
    *   **恶意站点部署:**
        - 建立水坑攻击 (Watering Hole) 站点，将利用代码部署在目标人员常访问的网站上。

### 阶段三：漏洞利用与植入 (Exploitation & Installation)
1.  **3.1 初始访问权限获取**
    => 执行投送操作，触发漏洞。
    *   向目标发送钓鱼邮件或引导其访问恶意站点。
    *   启动监听器 (Listener) 等待来自载荷的反向连接 (Reverse Shell)。

2.  **3.2 持久化后门安装**
    => 确保在目标重启或断开连接后仍能恢复访问。
    *   **Windows:** 写入注册表 (Run keys)、创建计划任务、安装为服务。
    *   **Linux:** 添加 `cron` 定时任务、创建 `systemd` 服务、修改启动脚本。
    *   安装功能更全的后门或Rootkit。

### 阶段四：指挥与控制 (Command & Control - C2)
1.  **4.1 C2通信链路建立**
    => 通过已植入的后门与目标系统建立稳定、加密的通信。
    *   使用C2框架如 `Cobalt Strike`, `Empire`, `Metasploit`。
    *   通过伪装流量 (Domain Fronting, DNS over HTTPS) 绕过网络检测。

2.  **4.2 权限提升与横向移动**
    *   **权限提升 (Privilege Escalation):**
        - 在目标系统上运行`LinPEAS`, `WinPEAS` 等脚本，查找本地提权漏洞或配置错误。
        - 利用内核漏洞或滥用SUDO/SUID权限，获取root/SYSTEM权限。
    *   **横向移动 (Lateral Movement):**
        - 从当前主机窃取凭据 (Mimikatz, LaZagne)。
        - 使用窃取的凭据通过 `PsExec`, `WMI`, `SSH` 等方式访问网络内其他主机。
        - 目标：获取域控权限。

### 阶段五：目标行动 (Actions on Objectives)
1.  **5.1 数据发掘与窃取 (Data Exfiltration)**
    *   在内部网络中定位、打包和压缩目标数据（如：数据库、文档、源代码）。
    *   通过隐蔽信道（如：ICMP, DNS隧道）将数据缓慢传输出去，避免触发流量异常警报。

2.  **5.2 系统破坏或勒索 (Destruction/Ransom)**
    *   **破坏:** 删除关键文件、破坏引导记录 (MBR)、造成逻辑或物理损坏。
    *   **勒索:** 使用勒索软件加密全网关键文件，留下勒索信息。

### 阶段六：清理与撤退 (Cleanup & Retreat)
1.  **6.1 清除痕迹**
    *   删除或修改系统日志、安全日志、应用日志 (`/var/log`, Windows Event Logs)。
    *   使用工具清除Shell历史记录。
    *   删除上传的所有工具和脚本。

2.  **6.2 维持或放弃访问**
    *   **维持:** 留下一个或多个高度隐蔽的休眠后门，以备将来访问。
    *   **放弃:** 彻底移除所有植入物，断开连接。

## 资源/角色详情

### 角色与职责
1.  **操作员 (Operator)**
    + **职责:** 规划、执行并管理攻击生命周期的所有阶段。要求具备跨领域的技术能力和极高的操作安全性。

### 关键资源清单
1.  **硬件平台 (Attack Platform)**
    + 一台专用的、物理隔离的计算机。
    - 使用加密的全盘存储 (LUKS, VeraCrypt)。
    - 推荐使用一次性硬件或难以追踪来源的二手设备。

2.  **软件与工具 (Toolkit)**
    + **操作系统:** Kali Linux, Parrot Security OS, 或一个经过硬化的、匿名的自定义Linux发行版。
    - **核心框架:** Metasploit, Cobalt Strike。
    - **扫描器:** Nmap, Masscan, Burp Suite, Nessus。
    - **密码工具:** John the Ripper, Hashcat, Mimikatz。
    - **匿名化:** Tor, Whonix, Tails。

3.  **基础设施 (Infrastructure)**

    - **C2服务器:** 租用Bulletproof Hosting（防弹主机）或已被攻陷的第三方服务器作为跳板。
    - **域名:** 使用匿名方式注册的域名，用于钓鱼和C2回调。

4.  **信息资产 (Intel)**
    + 目标的技术栈、人员信息、网络拓扑图。
```

```markdown
# OPSEC 详细文档

## 安全领域详情

### 数字安全 (Digital Security)
1.  **身份匿名**
    *   **IP地址:** 绝对禁止直接连接。必须使用多层代理，如 `VPN -> Tor` 或 `VPN -> a series of Proxies`。
    *   **身份信息:** 使用假名、一次性邮箱、加密货币 (Monero) 进行所有线上活动和购买。
    *   **浏览器指纹:** 使用 `Tails` 或配置浏览器以抵抗指纹识别。

2.  **数据隔离与加密**
    *   **物理隔离:** 攻击平台绝不用于任何日常活动，且永不连接不信任的网络。
    *   **逻辑隔离:** 使用虚拟机 (VM) 进行不同阶段的操作。例如，一个VM用于侦察，一个VM用于武器化。事后可安全销毁。
    *   **全盘加密:** 攻击平台和所有存储介质必须全盘加密。
    *   **数据传输:** 所有跨网络传输的数据必须使用强加密 (PGP/GPG)。

3.  **操作环境无菌化**
    *   **操作系统:** 使用非持久性操作系统如 `Tails`，重启后不留痕迹。
    *   **MAC地址:** 每次联网前都随机化网卡MAC地址。
    *   **虚拟机快照:** 在执行高风险操作前创建VM快照，一旦出错或被发现，可立即恢复到干净状态。

### 基础设施安全 (Infrastructure Security)
1.  **C2架构韧性**
    *   **分层与重定向:** 使用重定向器 (Redirector) 作为C2服务器的前置跳板。即使跳板被发现，核心C2服务器依然安全。
    *   **可抛弃性:** 所有基础设施（服务器、域名）都应被视为一次性的，随时准备抛弃。

2.  **网络流量混淆**
    *   **C2 Profile:** 修改C2框架的流量特征 (Malleable C2 Profiles)，使其看起来像正常的Web流量、DNS查询或社交媒体流量。
    *   **Domain Fronting:** 将C2流量伪装成对高信誉域名（如Google, Cloudflare）的访问。

3.  **资产获取匿名化**
    *   **服务器/VPS:** 使用被盗信用卡或加密货币从无视滥用投诉的托管商处购买。
    *   **域名:** 使用匿名注册服务或在隐私保护严格的国家注册。

### 个人安全 (Personal Security)
1.  **物理环境隔离**
    *   操作地点应远离住处，选择无监控、人员流动复杂的公共场所。
    *   绝不将攻击设备与个人设备（如手机）放在一起。

2.  **行为准则与纪律**
    *   **沉默:** 绝对禁止在线下或线上吹嘘、分享任何行动细节。
    *   **一致性:** 保持虚假身份的一致性，避免行为模式的突然改变。
    *   **情绪控制:** 避免因贪婪、愤怒或恐慌而做出非理性决策，严格遵守计划。

3.  **信息隔离**
    *   将行动知识与个人生活知识完全分开。
    *   不使用任何与个人身份相关的词汇、密码或习惯。

## 威胁/对策详情

### 主要威胁
1.  **侦测 (Detection)**
    => **风险:** 防火墙、IDS/IPS、EDR、SIEM系统记录并告警了你的活动。

2.  **溯源 (Attribution)**
    => **风险:** 安全研究员或执法部门通过技术或非技术手段将你的线上活动关联到你的真实身份。

3.  **阻断 (Interdiction)**

### 核心对策
1.  **预防性措施 (Proactive)**

    *   **免杀测试:** 在发起攻击前，在本地隔离环境中反复测试载荷，确保能绕过主流杀毒软件。

2.  **响应性措施 (Reactive)**
    *   **紧急撤离脚本:** 准备一个一键执行的脚本，用于立即删除关键文件、关闭连接、销毁虚拟机。

3.  **反取证措施 (Anti-Forensics)**

    *   **时间戳修改:** 修改你接触过的文件的时间戳 (timestomping)，使其与系统正常文件一致，迷惑取证分析。
    *   **内存驻留:** 尽可能使载荷仅在内存中运行，避免在磁盘上留下文件痕迹。
```
