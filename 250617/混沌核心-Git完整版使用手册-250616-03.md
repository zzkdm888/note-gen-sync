# Git 完整使用手册（包含SSH部分）

## 1. Git 安装与配置

### 1.4 SSH 配置（新增）

#### 1.4.1 生成SSH密钥

```bash
# 推荐使用Ed25519算法
ssh-keygen -t ed25519 -C "your_email@example.com"

# 或者使用RSA算法（兼容旧系统）
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

- 按提示操作：
  1. 输入保存路径（默认~/.ssh/id_ed25519）
  2. 设置安全密码（建议）
  3. 确认密码

#### 1.4.2 管理SSH代理

```bash
# 启动SSH代理
eval "$(ssh-agent -s)"

# 添加密钥到代理（macOS可添加--apple-use-keychain）
ssh-add ~/.ssh/id_ed25519

# 查看已加载密钥
ssh-add -l

# 删除特定密钥
ssh-add -d ~/.ssh/id_ed25519
```

#### 1.4.3 多账户SSH配置

```bash
# ~/.ssh/config 示例配置
Host github.com-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal
  IdentitiesOnly yes

Host github.com-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work
  IdentitiesOnly yes

# 使用方式
git clone git@github.com-personal:username/repo.git
git clone git@github.com-work:company/repo.git
```

#### 1.4.4 权限设置

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/config
chmod 600 ~/.ssh/id_*
chmod 644 ~/.ssh/*.pub
```

## 2. Git 基础操作

### 2.4 SSH相关操作（新增）

#### 2.4.1 测试SSH连接

```bash
ssh -T git@github.com
# 成功响应: "Hi username! You've successfully authenticated..."

# 详细调试模式
ssh -vT git@github.com
```

#### 2.4.2 转换远程URL为SSH

```bash
# 查看当前远程URL
git remote -v

# 修改为SSH格式
git remote set-url origin git@github.com:user/repo.git
```

## 4. 远程仓库操作（SSH补充）

### 4.3 SSH密钥管理（新增）

#### 4.3.1 添加公钥到Git服务

```bash
# 复制公钥内容
# macOS
cat ~/.ssh/id_ed25519.pub | pbcopy

# Linux
cat ~/.ssh/id_ed25519.pub | xclip -sel clip

# Windows
clip < ~/.ssh/id_ed25519.pub
```

然后粘贴到：

- GitHub: Settings → SSH and GPG keys → New SSH key
- GitLab: Preferences → SSH Keys
- Bitbucket: Settings → SSH keys

#### 4.3.2 密钥轮换最佳实践

1. 生成新密钥对
2. 将新公钥添加到所有相关服务
3. 测试新密钥正常工作
4. 从服务中删除旧公钥
5. 从本地~/.ssh/known_hosts中删除旧记录（如有需要）

## 5. 高级操作（SSH补充）

### 5.4 SSH相关问题排查（新增）

#### 5.4.1 常见错误解决

```bash
# 错误: Permission denied (publickey)
# 解决方案:
1. 确认ssh-agent正在运行
2. 确认密钥已添加: ssh-add -l
3. 确认公钥已正确添加到Git服务
4. 检查~/.ssh/config配置

# 错误: Host key verification failed
# 解决方案:
ssh-keygen -R github.com  # 清除旧主机密钥
```

#### 5.4.2 强制使用特定密钥

```bash
# 临时指定密钥
GIT_SSH_COMMAND="ssh -i ~/.ssh/specific_key" git clone git@github.com:user/repo.git

# 针对特定仓库永久设置
git config core.sshCommand "ssh -i ~/.ssh/specific_key"
```

## 8. 常见问题解决（SSH补充）

### 8.4 SSH相关问题（新增）

#### 8.4.1 修复SSH连接问题

```bash
# 完整诊断流程:
1. 测试基础连接: ssh -vT git@github.com
2. 检查密钥加载: ssh-add -l
3. 验证配置文件: cat ~/.ssh/config
4. 检查公钥匹配: 比较本地公钥和服务端记录
5. 检查known_hosts: cat ~/.ssh/known_hosts | grep github.com
```

#### 8.4.2 多平台SSH配置同步

```bash
# 安全地同步SSH配置:
1. 只同步~/.ssh/config和密钥文件
2. 使用加密方式传输密钥
3. 在不同设备上使用不同密钥对更安全
4. 考虑使用密码管理器存储加密备份
```

## 附录：SSH安全最佳实践

1. **密钥保护**

   - 始终为密钥设置密码
   - 使用ssh-agent管理密钥密码
   - 定期轮换密钥（建议每6-12个月）
2. **算法选择**

   - 优先选择Ed25519算法
   - RSA至少使用4096位
   - 禁用不安全的SSHv1和旧算法
3. **访问控制**

   - 不同服务使用不同密钥对
   - 使用~/.ssh/config管理多配置
   - 限制密钥的使用范围（GitHub允许添加部署密钥）
4. **审计与监控**

   - 定期检查Git服务的SSH密钥列表
   - 监控Git服务的访问日志
   - 使用 `ssh-keygen -l -f keyfile`验证密钥指纹
5. **备份策略**

   - 安全备份~/.ssh/config和密钥
   - 存储加密的密钥备份
   - 记录密钥对应的服务和用途
